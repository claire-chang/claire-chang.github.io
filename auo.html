JUL - AUG 2020・UX RESEARCH・DESIGN

AUO UX Design Internship

Collaborate with a team of engineers to reimagine work order dispatch system
across web and mobile platform for solar energy maintenance team.

Note: This is an NDA limited case study.

My role in the project
  • Conduct user research for maintenance dispatch software system and tranlate insights into design concepts and solutions.
  • Collaborate with a team of developers to iterate wireframes and prototypes.
  • Unify and define design guidelines and components such as data table.
Type

8 week internship

Mentor

Donald Chou, Miranda Chou

。 Background 。 Research 。 Design 。 Takeaways
BACKGROUND

Dispatch System transforms the traditional maintenance management system into a digitalized one, as the company aims to implement ML models to improve the efficiency of the maintenance work.
However, users, AUO's maintenance team, are experiencing a hard time to adapt to a new mindset and platform.

How can we make the whole process more intuitive to different kinds of users while meeting the bussiness goals?

RESEARCH

Before designing solutions for user, we needed to understand the user requirements and current workflow. Therefore, I conducted stakeholder interview to define main tasks and the stakeholder map.

Also, I leveraged cognitive walkthrough and comparative analysis to identify usability issues and potential strengthness and weakness within the workflow.

Main findings and design opportunities

  1. Difficulty to understand the workflow which the work item is required to add into the work order manually.
  2. Heavy cognitive load to users when assigning the work orders because the inability to reference the schedule.
  3. Disparity between envisioned and current usage of mobile platform where users need to more freedom of control such as import unsolved work items.
DESIGN APPROACH

Leverage digital product’s strengths to help users complete main tasks

Based on research insights, I proposed data-driven design concepts focusing on the main tasks that affects users the most, including the process of assigning orders, the process of adding items to orders, and the ability to select unsolved items on mobile app.

I started from sketches, and iterated wireframes and mockups after discussing with the team.

FINAL DESIGN EXAMPLE

Concept 1: Assign and Schedule Work Order

Dispatchers assign and schedule work orders to all the executors spreading out on different sites across the country.

Current and proposed design work flow

Users are not using the current system (ideal design) to complete this task, instead, they develop their own ways.
By identifying the burden of the ideal process, the proposed design provides information users need to eliminate cognitive overload. At the same time, users are able to do multi tasks on a single page.

Design Outcome

HYPOTHESIS

If dispatchers can assign the work order while having the access to the schedule of executors at the same time, it reduces dispatchers' memory load and dispatchers do not need to switch between tools.


EXPECTATION

If there are more work orders assigned through website than through mobile App, the hypothesis is true.




WHAT'S MORE...

I defined from the most common design components in the system to improve efficiency for development.




Takeaways
